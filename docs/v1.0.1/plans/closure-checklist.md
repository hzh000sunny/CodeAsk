# v1.0.1 Closure Checklist

> 用于记录 v1.0.1 从 `In Progress` 切换为 `Completed` 的关闭条件，以及哪些事项被正式后移。

## 1. 目的

v1.0.1 当前已经完成独立 Wiki 工作台的主链路，但文档口径仍存在两个事实并存：

- 主功能清单大多已经通过；
- 版本 README 曾标记为 `In Progress`。

本清单的职责不是重开新范围，而是给当前版本一个明确的关闭标准，回答两个问题：

1. 还有哪些事项必须在 v1.0.1 内收口？
2. 哪些事项虽然重要，但必须正式后移，不能继续挤进 v1.0.1？

## 2. 当前判断

截至 2026-05-06，v1.0.1 的状态定义如下：

- **主链路状态：已基本完成**
  - 独立 Wiki 页面、目录树、阅读态、编辑态、版本、导入会话、报告投影、搜索、排序、恢复/治理尾项已具备第一版可用能力。
- **版本状态：已封板**
  - 来源治理、真实浏览器回归和文档口径统一已经完成版本收口。

结论：

> v1.0.1 现已作为“边界清楚、文档一致、测试通过”的已完成版本关闭。

## 4. 明确后移，不再计入 v1.0.1 的事项

以下事项必须保留接口或文档通道，但不应继续算进 v1.0.1 收口范围。

### 4.1 权限体系恢复

不纳入 v1.0.1：

- `owner / admin / member` 写权限恢复
- 真实登录鉴权后的细粒度 Wiki 写限制
- 完整权限审计与访问隔离

原因：

- 当前版本已明确覆写为“全员可写”。
- 再把权限体系塞回 v1.0.1，会重新冲击会话、特性、设置和 Wiki 的行为面。

### 4.2 AuthProvider 接入

不纳入 v1.0.1：

- OIDC / LDAP / 企业 IM / 统一账号体系
- 用真实身份替换当前 `subject_id + admin cookie`

原因：

- 这是跨系统能力，不属于 Wiki 专项尾项。

### 4.3 非 Markdown 文档格式

不纳入 v1.0.1：

- PDF / DOCX 直接导入
- 自动转换为 Markdown 的文档处理流水线

原因：

- 当前 PRD 已明确：知识库第一版只支持 Markdown。

### 4.4 更重的企业级来源能力

不纳入 v1.0.1：

- 外部连接器型来源
- 大规模来源批量修复
- 跨空间来源治理策略

原因：

- 当前来源治理第一版目标是“可追溯、可同步、可解释”，不是企业级来源平台。

### 4.5 LLM Agent 进一步优化

不纳入 v1.0.1：

- 更强的自然语言范围理解
- 更复杂的多目录组合推理
- 纯知识库问答与代码调查之间的更深运行时策略优化

原因：

- 用户已明确要求先收尾 Wiki 模块，把 agent 优化推迟到后续版本。
- 这些能力会影响会话运行时策略，不应继续阻塞 v1.0.1 的 Wiki 封板。
