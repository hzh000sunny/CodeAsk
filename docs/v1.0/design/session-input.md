# 会话与输入设计

## 1. 目标

会话系统负责接收用户问题、日志、附件和上下文，并把它们转成 Agent 可消费的结构化输入。

## 2. 会话原则

- 会话不预绑特性。
- 用户可以手动选择特性。
- 未选择时 Agent 自动定界。
- 会话可以跨多轮继续调查。
- 版本确认信息在会话内可复用。

## 3. 输入方式

一期支持：

- 文本问题。
- 粘贴日志。
- 上传日志文件。
- 可选上下文字段。
- 可选 feature 选择。
- 可选 repo ref / commit。

外部日志平台接入后置。

## 4. 附件

附件默认用于日志和文本类材料。

限制：

- 二进制文件拒绝。
- 单个日志默认上限 10MB。
- 附件归属当前 session。
- 24h 未活跃清理时删除附件文件，但保留 DB 记录。

## 5. 版本确认交互

当 Agent 需要查代码但缺少版本时：

- 可先默认分支预查。
- 如果预查不足，触发版本确认。
- 用户可输入 branch、tag、commit 或构建号。
- 系统解析并绑定到会话仓库上下文。

## 6. ask_user

`ask_user` 是 Agent 主动追问机制。前端应以明确问题和选项展示，并把用户回答写回当前会话。

追问次数默认单轮最多 3 次。
